COMMANDS

Four commands.
Full attendance cycle.

No forms, no dashboards — employees track their entire workday without leaving Slack. Every action syncs instantly to Odoo's HR Attendance module.

/login

Check In

Records attendance in Odoo. Grace minutes applied automatically.

/break

Start Break

Begins a timed break. Short breaks below the threshold are saved but not counted.

/resume

End Break

Returns to active work. Break duration is automatically evaluated against the minimum.

/logout

Check Out

Closes the attendance record. Net working hours are computed and stored.

FEATURES

Everything you'd actually need.

Built for real workplaces — not just demos. Every edge case is handled.

⏱

Login Grace Time

Configure a grace window (e.g. 5 min) so an employee who types /login at 10:00 AM is recorded as 9:55 AM. No more clock-racing anxiety.

☕

Break Threshold

Quick bathroom breaks below the minimum threshold are saved but not deducted from working hours. Only meaningful breaks count.

🧮

Net Working Hours

Every record shows: Net Hours = Total − Counted Breaks. Transparent and accurate — always.

🌍

Per-Employee Timezone

No hardcoded timezones. Each employee's timezone flows from their HR record → company → IST fallback.

🔔

Missed Logout Alerts

Employees who forgot to /logout get an automatic DM reminder. No ghost attendance records ever again.

🤖

AI Daily Summaries

Optional personalised DMs powered by Gemini (free) or Claude. References actual hours, streaks and targets — not a generic template.

SETUP

Up in under 10 minutes.

Four steps and you're live. No cloud subscription. No vendor lock-in.

01

Install the Module

Copy the folder to your Odoo addons path, restart, and install from Apps.

02

Configure Slack App

Create 4 slash commands in Slack, all pointing to /slack/attendance on your Odoo domain.

03

Enter Credentials

Add your Signing Secret, Bot Token, and Webhook URL in Attendances → Configuration → Slack Settings.

04

Link Employees

Open each HR employee record and paste their Slack Member ID. Done — they can start using commands instantly.

✨ OPTIONAL AI FEATURE

Personalised summaries.
Not robot templates.

Enable AI summaries with a Gemini (free) or Claude API key. Each employee receives a unique end-of-day message referencing their real hours, daily streak, and gap from target. The API key is stored in an admin-only field — no one else sees it.

CONFIGURATION

Every knob is yours.

Configure from Attendances → Configuration → Slack Settings.

FIELD DEFAULT DESCRIPTION
login_grace_minutes 5 Subtracts N minutes from actual login time — removes clock-racing pressure.
min_break_minutes 5 Breaks shorter than this are saved but not deducted from net working hours.
min_working_hours 8.0 Daily hour target used in summary comparison and AI prompt context.
send_daily_summary True Toggle end-of-day Slack DMs on or off globally.
summary_time 18.0 24-hour time when the daily cron fires (e.g. 18.0 = 6:00 PM).
ai_enabled False Enable AI-powered personalised summaries instead of the plain template.
ai_provider gemini Choose between Google Gemini (free) or Anthropic Claude.
ai_api_key — API key stored in an admin-only field group. Never exposed to regular users.
SECURITY

Enterprise-grade.
Zero compromises.

🔐

HMAC-SHA256 Signature Verification

Every inbound Slack request is verified with your Signing Secret before any database action is performed.

🛡

Replay Attack Protection

Requests older than 5 minutes are automatically rejected. HMAC's compare_digest prevents timing attacks.

🔒

certifi SSL — No CERT_NONE

All outbound HTTPS uses the certifi CA bundle. No certificate verification bypass, ever.

👁

Admin-Only API Key Storage

The AI API key field is gated behind base.group_system — invisible to all non-admin users.
